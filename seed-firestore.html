Setup Instructions

Before running the seed, you need to:

  1. Create a Firebase project at console.firebase.google.com
  2. Enable Firestore Database (production mode)
  3. Get your web app config (Project settings → Your apps → web </>)
  4. Edit firebase-config.js and paste your config values
  5. Set window.SIERRA_FIREBASE_ENABLED = true in that file
  6. Deploy firestore.rules (via Firebase CLI or console)
  7. Refresh this page — the status above should turn green

⚠️ Important: After seeding, also create a users document for yourself with role: "admin" — otherwise no one can write to the database (rules require admin for writes).

Run this in the browser console after seeding:

firebase.firestore().collection('users').doc('YOUR_AUTH_UID').set({
  email: 'your@email.com',
  role: 'admin',
  name: 'Ahmed'
});

After Seeding

  • The SE portal will automatically use Firestore (via firebase.js)
  • If Firestore is unavailable, it falls back to data.js (always works)
  • Inquiry form will write to inquiries collection (with localStorage fallback)
  • To enable Firestore on the live site, add the Firebase scripts to index.html (see README)
